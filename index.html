Render Methods

The first option you can choose that affects this info panel here are the render methods

The Render Method is how the visual window below will be drawn.

Using either a slow and hacky per pixel draw method made of tons of HTML/CSS Grid Cells or by using a proper drawing method like WebGL or WebGPU

What's the difference?

HTML/CSS Grids

This isn't a real method of drawing things to the screen, no one should ever do this for any real project. This method is slow, inefficient and for the developer, requires a lot more work.

WebGL

This method is similar to OpenGL, a Graphics API used to draw things to a real application window instead of a HTML Canvas.

It handles many things from turning a set of points into real shapes to handling the transfer of data between the computer and Graphics Card.

api

WebGPU

This method is similar to Vulkan, which to OpenGL is the modern alternative but not replacement.

Vulkan and subsequently WebGPU is another Graphics API but provides more control to the developer instead of automatically handling features.

Upsides to more control is that the developer can choose not to activate all features, increasing performance

Downsides is that there is more to program and develop

vulkan

Graphics Cards

For any of the prior two Graphics APIs to work, you will need a compatible graphics processor

Modern Computers will have a portion of their computing power dedicated to graphics either as a separate card or integrated into the CPU

If you can see things on a monitor, chances are you have a functional graphics processor

Browser Feature Sets

Having modern computer only guarantees you to be able to run the non-web counterparts of the Graphics APIs

You will also need a browser that supports the said Graphics APIs

Vertices

Vertices present a point in space, multiple vertices joined together will form a shape.

Models will usually have hundreds of Vertices.

Matrices

Vertices of a model usually have fixed positions. Static models aren't very exciting though. To move models you will need to move the vertices.

Translating is pretty simple. However, when rotating and scaling, things get pretty complicated.

Matrices let us hold all these transforms in one variable.

Model Matrix

The model matrix manipulates the model into their positions, rotations and scale

Projection Matrix

The projection matrix manipulates vertices such that when they are rendered, it can appear as if it has Perspective shifts or in the case of orthogonal, to allow for the Camera shifts.

View Matrix

The view matrix represents the virtual camera that the GPU will render through. Combined with the perspective matrix, we can simulate a Camera in our virtual world.

MVP Matrix

The MVP matrix combined the Model/View/Projection Matrices into one matrix, it lets us calculate the final point of the vertices.
